HTML 표기법 설명

CSS 표기법

CSS는 직접 입력하지 않고 "SASS를 이용하며 개발자분들께서 수정이 필요할 경우 "developer.css"를 이용

CSS 표기법 설명
CSS 기본
  • "인라인태그"로 작성은 자제
  • 속성과 값 사이에 한칸 띄워서 작성(예: width: 0;)
CSS 주석 
/* Contents */
Hex 색상코드 사용 color: #000000;
숫자 0 top: 0; (px등 단위를 입력하지 않음)

SASS 표기법

SASS 표기법 설명
SASS 기본 SCSS 를 사용
SASS 주석 
// *----------------------------
//	Contents
// ----------------------------*

제목 가이드라인

제목은 H태그를 사용해야 하나 기존사이트 자체에서 구분짓지 않아 클래스로 구분

h1 태그

대부분 사이트와 동일하게 상단 로고에 지정 (h1은 페이지에 유일해야 함)

h2 태그

class="title"

h3 태그

  • 고객페이지: class="subheading"
  • 관리자페이지: class="content-title"
  • 해당 상황은 3곳에 사이트를 가져오다 보니 발생한 상황으로 차후 통일할 예정

블릿 가이드라인

H4태그 부터는 사용을 안하며 필요시 블릿으로 구분

버튼 1개일때

Button 사용법 설명
설명
  • 클래스명 "btn-row"로 버튼 영역을 확보
  • 버튼영역 클래스는 상황에 따라 다를수 있음
  • 클래스명 : "one-btn"
  • 테이블이 아닌 일반적인 페이지 버튼에 사용 (200 * 50)
  • 클래스명 : "admin-btn"
  • 버튼 영역에 "has-one"을 추가하여 오른쪽 정렬
  • 페이징이 있는 테이블에 사용하며 버튼 1개일때는 흰색 테두리 버튼(클래스명 white 추가) (120 * 50)
  • 일부 페이지는 아직 "tpms-btn"으로 되어 있음 (상황에 맞지 않아 변경할 예정) 현재 고객 페이지 버튼 한개짜리에서 일부 발견
  • 클래스명 : "search-btn"
  • 버튼 영역이 "btn-row"가 아닌 "search-box"이며 search box 안에 버튼을 넣어야 할 때 사용 (103 * 40)
  • 클래스명 : "lookup-btn"
  • 버튼 영역이 "btn-row"가 아닌 "form-btn-box"이며 테이블 내에 버튼을 넣어야 할 때 사용
  • 버튼 사이즈는 103 * 40 이나 form-btn-box 클래스가 "display: flex"라 크기는 변동
  • 클래스명 : "popup-one-btn"
  • 팝업에 사용되는 버튼이 1개일때 사용 (103 * 40)
